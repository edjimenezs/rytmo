  const candidate = dateString ? new Date(dateString) : new Date();
  if (Number.isNaN(candidate.getTime())) throw new Error('Invalid date format');
  candidate.setUTCHours(0, 0, 0, 0);
  return candidate;
}

Create `app/api/feedback/trends/route.ts` with a single GET handler.

Implementation:
1. Import `NextRequest`, `NextResponse` from `next/server`
2. Import `prisma` from `@/lib/prisma`
3. Import `requireAuth` from `@/lib/auth/utils`
4. Import `subDays`, `startOfDay`, `format` from `date-fns`
5. Import `es` locale from `date-fns/locale/es`

GET handler logic:
- Call `requireAuth()` to get user (throws 'Unauthorized' on failure)
- Read `days` query param: `req.nextUrl.searchParams.get('days')`
- Clamp days: `Math.min(Math.max(parseInt(daysParam ?? '7', 10), 3), 14)`
- Compute `since = startOfDay(subDays(new Date(), days))`
- Query: `prisma.dailyFeedback.findMany({ where: { userId: user.id, date: { gte: since }, energy: { not: null } }, orderBy: { date: 'asc' }, select: { date: true, energy: true, performance: true } })`
- Map results: `feedbacks.map(f => ({ date: format(new Date(f.date), 'EEE d', { locale: es }), energia: f.energy, performance: f.performance }))`
- Return: `NextResponse.json({ trends, count: trends.length })`

Error handling (same pattern as feedback/route.ts):
```typescript
catch (error) {
  const message = error instanceof Error ? error.message : 'Unauthorized';
  const status = message === 'Unauthorized' ? 401 : 500;
  return NextResponse.json({ error: message }, { status });
}
```

Important: Do NOT use Prisma groupBy -- DailyFeedback already has `@@unique([userId, date])` so there's exactly one record per day. A simple `findMany` with date range filter is correct.
